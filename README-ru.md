## О проекте

Это настольное приложение для общения с моделями Mistral. Никаких сторонних сервисов: только Mistral, ключ хранится локально через `safeStorage` и не уходит никуда, кроме официального API.

**Чат** Ответы приходят по SSE и рисуются токен за токеном с живым курсором. Поддерживается markdown в стиле GitHub, подсветка синтаксиса и кнопки «копировать» в блоках кода. Автоскролл умный: пауза, как только ты прокрутил вверх. Генерацию можно прервать в любой момент, рядом считается расход токенов.

**Настройки** Две колонки: API, модель, вывод, интерфейс, горячие клавиши. Кастомные слайдеры для temperature и top-p, тест соединения с замером задержки, темы Dark / OLED / Dim и размер шрифта - всё применяется сразу, без перезапуска.

**История** Плотная таблица сохранённых сессий с поиском, выделением строк, массовым экспортом и удалением. Любую сессию можно открыть или выгрузить в `.md` либо `.json`.

**Системный промпт** Редактор с моноширинным шрифтом, нумерацией строк и счётчиком символов и токенов. В комплекте пресеты: General, Code Assistant, Translator, Analyst.

**Командная палитра** (`Ctrl+K`) и горячие клавиши по всему приложению.

### Стек

```
src/
├── main/
│   ├── main.js      Electron main: безрамочное окно, сохранение состояния окна,
│   │                IPC для electron-store, ключ через safeStorage, экспорт
│   ├── preload.js   Контекстный мост - единственная точка IPC (contextIsolation)
│   ├── mistral.js   Сервис Mistral API: нативный fetch, SSE-стриминг, abort
│   ├── agent.js     Агентный цикл: вызовы инструментов, гейты прав,
│   │                планирование/подтверждение, изолированные субагенты
│   ├── tools.js     Реестр инструментов - схема, права и реализация в одной таблице
│   ├── consoles.js  Постоянные shell-сессии (состояние живёт между командами)
│   ├── skills.js    Скиллы: поиск, парсинг и рендер Markdown-плейбуков
│   └── skills/      Встроенные скиллы (debug · review · simplify · analyze · …)
└── renderer/
    ├── index.html   Оболочка приложения (титлбар + навигация + контент)
    ├── app.js       Роутер страниц, шорткаты, командная палитра
    ├── shared.js    Хелперы стора, тосты, модалки, рендер markdown
    ├── models.js    Каталог моделей и их возможности
    ├── pages/       chat · settings · history · system-prompt · about
    ├── styles/      base · layout · components (CSS-переменные)
    └── vendor/      собранные marked + highlight.js (бандлятся через esbuild)
```

Безопасность: `contextIsolation: true`, `nodeIntegration: false`, строгий CSP. Весь доступ к Node и API идёт через именованные IPC-каналы (`store:*`, `apikey:*`, `mistral:*`, `window:*`, `session:*`).

## Установка

Скачай готовую сборку со страницы релизов:
#### [Последний релиз](https://github.com/Thonny-Developer/mist-desktop/releases/latest)

Там лежат установщик и портативная версия. После запуска открой **Настройки** и выбери пункт **API**, вставь ключ Mistral и нажми **Test connection**

### Сборка из исходников

```bash
npm install        # загрузка библиотек
npm start          # запуск
npm run dev        # запуск с DevTools
```

Сборка дистрибутивов:

```bash
# NSIS-установщик + портативный .exe в dist/
npm run dist:win

# сборка под Linux
npm run dist:linux
```

Под Windows можно заранее положить `build/icon.ico` (см. `build/README.md`)

## Как работают агенты

Агент - это не отдельная команда, а режим работы чата. Модель получает в контекст каталог инструментов и действует циклом: думает, вызывает инструмент, видит результат, решает, что делать дальше. Так продолжается, пока она не выдаст финальный ответ

Один проход выглядит так:

1. Ты пишешь задачу в чат
2. Модель отвечает обычным текстом и/или вызывает инструмент через блок `<action>`
3. Вызов парсится и выполняется в main-процессе
4. Результат возвращается модели как новое сообщение
5. Цикл повторяется - максимум 10 шагов с инструментами на один запрос

Действие модель выражает так:

```
<action>{"tool":"write_file","path":"src/app.js","content":"…"}</action>
```

Вся активность агента видна прямо в чате: какой инструмент вызван, с какими аргументами и что вернул. Выбранная рабочая папка и чип `todos N/M` живут в шапке чата

### Права и подтверждения

У каждого инструмента есть метка прав (`readOnly`, `mutating`, `destructive`, `shell`, `web`). В обычном режиме изменяющие и опасные действия требуют подтверждения, а чтение проходит свободно. Агент может сперва показать план и дождаться одобрения, прежде чем что-то менять

### Скиллы

Скилл - это переиспользуемый Markdown-плейбук с YAML-шапкой (`name`, `description`, `allowed-tools`, `argument-hint`). Скиллы ищутся в трёх слоях: встроенные → `<userData>/skills` → `<workingDir>/.mist/skills`, причём поздний слой переопределяет ранний

Запустить скилл можно слэш-командой в чате (`/review src`), либо модель сама выберет его через `run_skill` / `list_skills`. В тело подставляются `$ARGUMENTS` и `$1…$9`, а сам скилл может ограничить набор доступных инструментов своим `allowed-tools`. Управление в **Настройки** пунтк **Skills**

```markdown
---
name: review
description: Review recent changes for bugs and improvements
allowed-tools: read_file, list_files, exec_bash
argument-hint: [path or focus]
---
You are a senior reviewer. Review $ARGUMENTS …
```

### Субагенты

Через инструмент `run_subagent` агент делегирует подзадачу изолированному воркеру. У того свой контекст и урезанный, в основном read-only набор инструментов. Субагент работает сам, без вопросов к пользователю, ограничен по глубине вложенности и возвращает один текстовый отчёт, который становится результатом инструмента для родителя. Его работа тоже видна в чате - отступом под основными строками

## Как работают инструменты

Все инструменты объявлены в одной таблице - реестре `tools.js`. Каждый инструмент описан там ровно один раз: и схема для вызова моделью, и метаданные прав, и реализация

Из этой таблицы выводится сразу всё: схемы, которые уходят модели, гейты прав в цикле агента, список для обнаружения инструментов и тот read-only поднабор, который разрешён субагенту. Поэтому инструмент не может «разъехаться» между тем, как он зарегистрирован, описан и ограничен в правах

### Доступные инструменты:
> В список инструментов будет часто обновляться

| Инструмент | Что делает |
|---|---|
| `set_working_folder` | Выбрать рабочую папку - песочницу для файловых операций |
| `list_files` | Показать содержимое папки |
| `read_file` | Прочитать файл |
| `write_file` | Записать или создать файл |
| `delete_file` | Удалить файл |
| `add_todo` | Добавить задачу в список |
| `complete_todo` | Отметить задачу выполненной |
| `list_todos` | Показать текущие задачи |
| `remember` | Записать факт в долгую память |
| `run_skill` / `list_skills` | Запустить скилл или показать доступные |
| `run_subagent` | Делегировать подзадачу изолированному субагенту |

Все пути к файлам жёстко ограничены выбранной рабочей папкой - выйти за её пределы инструмент не может

### Vendor-бандл

`marked` и подобранный набор языков `highlight.js` собираются в `src/renderer/vendor/libs.js` через esbuild (ESM-вход highlight.js тянет CommonJS-нутро, которое браузерный загрузчик не съедает напрямую). Это происходит автоматически на `postinstall`, пересобрать вручную:

```bash
npm run build:vendor
```
